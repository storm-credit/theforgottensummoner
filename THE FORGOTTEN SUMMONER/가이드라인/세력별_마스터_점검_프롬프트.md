## 점검 지시 사항 (Deep Dive)
1. **개연성(Probability & Consistency) 크로스 체크 [가장 중요]**:
   - 세력의 이념, 정치, 경제적 특성이 대륙의 환경설정과 충돌하지 않는가?

2. **워크플로우 준수 여부 점검**:
   - 문서를 구성하는 형식(YAML 구조, 하이퍼링크, 필수 파라미터 등)이 Premium Quality Standard를 완벽히 따르고 있는가?
3. **확장성 및 깊이 보완 (Suggestions)**:

## 세부 검증 계층 (5 Layer Cross-Check)

### Layer A: 세력 본문
- 프리미엄 9섹션 양식 준수 (경제 섹션은 해당 세력에만)
- YAML frontmatter, 위키링크(`[[ ]]`), 필수 파라미터
- 대륙 환경설정과의 개연성 충돌, 타임라인 모순

### Layer B: 주요 인물
- 인물표 ↔ 캐릭터 파일 이름/등급/직위/거점 정합
- 인물 동기가 세력 핵심 가치와 맞물리는지

### Layer C: 성장 영웅
- 성장 동선이 세력 역사 타임라인과 모순 없는지
- 영웅 능력·장비가 세력 특성에 부합하는지

### Layer D: 아이템 로어 매핑 (N:N 다대다 관계)

#### D-1. 인물 → 아이템 (인물 파일 기준)
한 캐릭터가 여러 아이템을 보유할 수 있음:
- 검 + 갑옷 + 반지 (서로 다른 카테고리)
- 검 2자루 (같은 카테고리 복수)
- 무기만 / 방어구만 / 악세서리만 보유 가능
→ 인물 파일에 기재된 모든 아이템의 `[[위키링크]]`와 아이템 도감 파일 존재 여부 확인

#### D-2. 아이템 → 인물 (아이템 파일 기준)
- `현재 소유자` 필드가 `[[인물명]]`을 정확히 가리키는지
- 소유자 변경 시 `역대 주인` 이력 기록 여부
- 무소유 유물은 `소유자: none` 명시

#### D-3. 개연성 체크
- 아이템 제작 기원의 시간선이 세력 역사와 모순되지 않는지
- 아이템 등급 밸런스: 신화 > 전설 > 영웅 > 희귀 > 일반
- 세력 상징 유물/무기가 아이템 도감에 파일로 존재하는지

#### D-4. 누락 패턴 감지
- 인물 파일에 장비 섹션이 없는 캐릭터 목록
- 아이템 도감에 존재하나 인물에 링크되지 않은 고아 아이템
- 세력 문서에 언급되었으나 도감에 파일 없는 유령 아이템

### Layer E: 대외 관계
- A→B 우호이면 B→A도 우호인지 (양방향 정합)
- 유령 세력 참조 없는지
- 대륙 간 관계가 지리적으로 가능한지
